# `tokens/` — what is source, what is generated

This directory mixes the hand-maintained source of truth with files that are derived from
it or staged for the Figma sync. **Only edit the files marked as source.** Everything else
is regenerated by `npm run build` (and `check:dist` fails CI if it drifts).

| Path                                                                                              | Role                                                                                                                                                                                             | Edit by hand?                     |
| ------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------- |
| `{global,semantic}/color/canonical.json`                                                          | **Source of truth.** DTCG 2025.10 sRGB colour objects (`{colorSpace, components, alpha, hex}`).                                                                                                  | ✅ Yes                            |
| `themes/color/<theme>/canonical.json`                                                             | **Source of truth** per theme. Alias themes reference globals (`{nsw-blue.50}`); concrete themes hold sRGB objects.                                                                              | ✅ Yes                            |
| `*/color/{hex,rgb,hsl,oklch}.json`                                                                | **Generated** per-colour-space views, derived from `canonical.json` by `scripts/build-token-views.mjs` (hsl/oklch rounded to 6 dp for determinism).                                              | ❌ No — regenerate                |
| `primitives-global.light.json`, `primitives-semantic.light.json`, `themes-masterbrand.light.json` | **Figma sync staging.** Round-trip files for `sync-figma-to-tokens` / `sync-tokens-to-figma`; same DTCG sRGB shape. They mirror Figma variable collections (see `scripts/figma-collections.ts`). | ⚠️ Via the sync scripts, normally |

## Day-to-day

- **Change a colour:** edit the relevant `canonical.json`, then `npm run build` and commit
  the regenerated views, `src/`, and `dist/` together.
- **Add a theme:** create `themes/color/<theme>/canonical.json`, add it to
  `build/style-dictionary.config.mjs` and `scripts/build-token-views.mjs`, rebuild.
- **Never** edit `hex/rgb/hsl/oklch.json` directly — the next build overwrites them.

Validation: `npm run validate:tokens` checks DTCG conformance, alias resolution, duplicate
names, and value shape across this whole tree.

See [`docs/architecture.md`](../docs/architecture.md) for the full pipeline
(`canonical.json → views → Style Dictionary → src/ → tsup → dist/`).
