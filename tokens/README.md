# `tokens/` ‚Äî what is source, what is generated

This directory mixes the hand-maintained source of truth with files that are derived from
it or staged for the Figma sync. **Only edit the files marked as source.** Everything else
is regenerated by `npm run build` (and `check:dist` fails CI if it drifts).

| Path                                                                                              | Role                                                                                                                                                                                             | Edit by hand?                     |
| ------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------- |
| `{global,semantic}/color/canonical.json`                                                          | **Source of truth.** DTCG 2025.10 sRGB colour objects (`{colorSpace, components, alpha, hex}`).                                                                                                  | ‚úÖ Yes                            |
| `themes/color/<theme>/canonical.json`                                                             | **Source of truth** per theme. Alias themes reference globals (`{nsw-blue.50}`); concrete themes hold sRGB objects.                                                                              | ‚úÖ Yes                            |
| `*/color/{hex,rgb,hsl,oklch}.json`                                                                | **Generated** per-colour-space views, derived from `canonical.json` by `scripts/build-token-views.mjs` (hsl/oklch rounded to 6 dp for determinism).                                              | ‚ùå No ‚Äî regenerate                |
| `primitives-global.light.json`, `primitives-semantic.light.json`, `themes-masterbrand.light.json` | **Figma sync staging.** Round-trip files for `sync-figma-to-tokens` / `sync-tokens-to-figma`; same DTCG sRGB shape. ## Day-to-day

- **Change a colour:** edit the relevant `canonical.json`, then `npm run build` and commit
  the regenerated views, `src/`, and `dist/` together.
- **Add a theme:** create `themes/color/<theme>/canonical.json`, add it to
  `build/style-dictionary.config.mjs` and `scripts/build-token-views.mjs`, rebuild.
- **Never** edit `hex/rgb/hsl/oklch.json` directly ‚Äî the next build overwrites them.

## Raw values vs aliases ‚Äî deliberate, not an oversight

- **Semantic tokens (`success`/`warning`/`danger`/`info`) hold raw sRGB values, not aliases
  to global primitives.** This was audited during the transformer migration: the semantic
  palettes are independent ramps ‚Äî **0 of 19 steps** in any semantic family match a global
  primitive ‚Äî so aliasing them would either change shipped colours or point at the wrong
  intent. Don't "fix" them into aliases without a design decision to actually re-base the
  palettes.
- **Themes differ by design:** `masterbrand`, `fuchsia-blue`, and `fuchsia-orange` are
  alias themes (`{nsw-blue.50}` etc. ‚Äî they re-map global primitives), while
  `data-visualisation` is a concrete palette with raw values because its colours don't
  correspond to any primitive ramp.
- **`$description` lives in `canonical.json`** (semantic + masterbrand are fully described)
  and flows into the generated views and the CSS/SCSS/LESS outputs as per-variable
  comments. Keep descriptions updated when adding tokens ‚Äî the Figma sync round-trips them.

Validation: `npm run validate:tokens` checks DTCG conformance, alias resolution, duplicate
names, and value shape across this whole tree.

See [`docs/architecture.md`](../docs/architecture.md) for the full pipeline
(`canonical.json ‚Üí views ‚Üí Style Dictionary ‚Üí src/ ‚Üí tsup ‚Üí dist/`).
