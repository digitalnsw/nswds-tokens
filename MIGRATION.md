# Migration guide

`CHANGELOG.md` is generated by semantic-release from commits; this file collects the
human-facing migration notes for breaking releases.

## v3.0.0 — DTCG 2025.10 colour shape + single-source colours

The colour tokens are now generated from a single DTCG-compliant source. **CSS/hex/rgb
consumers are unaffected**; the breaking changes are in the raw token JSON and the
re-derived hsl/oklch values.

### What changed

- **Raw token object shape (`tokens/`, `dist/tokens/`, `figma/`)** now follows the
  [DTCG 2025.10 Color module](https://www.designtokens.org/tr/2025.10/color/):
  - `channels` → **`components`**
  - `colorSpace: "rgb"` → **`"srgb"`**, with components normalised **0–255 → 0–1**
  - a powerless (achromatic) hue is the string **`"none"`** (was `null`/`0`)
  - each colour object carries an optional **`hex`** fallback
- **Published hsl and oklch values changed.** They are now re-derived from the canonical
  sRGB value via culori, rounded to **6 decimal places** (full IEEE precision is not
  reproducible across platforms). **hex and rgb values are byte-identical** to before.
- The Primitives/Themes Figma-sync files (`primitives-*.light.json`,
  `themes-*.light.json`) use the same DTCG sRGB object shape (was a hex string).
- New theme colours are first-class: `data-visualisation`, `fuchsia-blue`,
  `fuchsia-orange` are now generated in every format and exposed on the root `tokens`
  export (additive).

### How to migrate

- **CSS / SCSS / LESS / hex / rgb consumers:** no change.
- **Consumers of `hsl`/`oklch` outputs:** values shift in the lower decimals; if you
  snapshot them, re-baseline.
- **Consumers of the raw DTCG JSON (`@nswds/tokens/tokens/*`, `/figma/*`) or
  `tokens.colors.<layer>.{hsl,rgb,oklch}`:** read `components` (not `channels`),
  `colorSpace: "srgb"` (not `"rgb"`), components in 0–1, and `"none"` for powerless hue.
  A `hex` fallback is available on every colour object.
